O método "pushState()" adiciona um novo estado à pilha de histórico do navegador. Quando você usa pushState, a URL na barra de endereços é atualizada, mas a página não é recarregada. Isso é útil para navegar entre diferentes seções de um SPA sem interromper a experiência do usuário com recarregamentos. O método replaceState() funciona de maneira semelhante ao pushState, mas em vez de adicionar uma nova entrada ao histórico, ele substitui a entrada atual. Isso é útil quando você quer alterar a URL sem adicionar uma nova entrada de histórico, evitando que o usuário possa voltar para uma URL anterior. O evento popstate é acionado sempre que o usuário navega para um estado diferente na pilha de histórico. Esse evento é disparado quando o usuário clica no botão "voltar" ou "avançar" do navegador. Você pode combinar essas funções para criar um sistema de rotas simples sem recarregar a página.
